**任务描述**:
封装 axios 实例，配置请求拦截器、响应拦截器，统一错误处理，支持 JWT 认证。

**核心功能**:
1. **axios 实例创建**: 基础 URL、超时时间、headers
2. **请求拦截器**: 
   - 自动添加 `Authorization: Bearer <token>` header
   - 请求日志（开发环境）
3. **响应拦截器**:
   - 统一处理响应格式 `ApiResponse<T>`
   - 错误码处理：401 跳转登录、403 提示无权限、404 提示资源不存在、500 提示系统错误
   - 业务错误处理：`code !== "0"` 时抛出错误
4. **错误处理**:
   - 网络错误提示
   - 超时错误提示
   - 业务错误提示（使用 Ant Design 的 message 组件）
5. **请求重试**（可选）: 网络错误自动重试

**验收标准**:
- [ ] `src/api/request.ts` 创建 axios 实例
- [ ] 请求拦截器自动添加 JWT token
- [ ] 响应拦截器统一处理 `ApiResponse<T>` 格式
- [ ] 错误处理覆盖：401、403、404、500、网络错误、超时
- [ ] 业务错误（`code !== "0"`）抛出错误并提示
- [ ] 401 错误自动跳转登录页
- [ ] 请求/响应日志（开发环境）
- [ ] 单元测试覆盖核心逻辑

**技术备注**:
- 使用 `axios.create()` 创建实例，不修改全局 axios
- JWT token 从 localStorage 或 zustand store 获取
- 错误提示使用 `antd` 的 `message` 组件
- 401 错误跳转登录页时，清除本地 token 和用户信息
- 响应拦截器返回 `response.data.data`（直接返回业务数据）

**任务描述**:
实现认证状态管理，包括 JWT 存储、登录状态、用户信息、权限信息。

**状态管理方案**:
使用 Zustand 或 Redux Toolkit 管理认证状态。

**状态结构**:
```typescript
interface AuthState {
  token: string | null;
  user: UserVO | null;
  permissions: string[];
  menus: MenuVO[];
  isAuthenticated: boolean;
  
  // Actions
  login: (token: string, user: UserVO) => void;
  logout: () => void;
  setUser: (user: UserVO) => void;
  setPermissions: (permissions: string[]) => void;
  setMenus: (menus: MenuVO[]) => void;
}
```

**JWT 存储**:
- 登录成功后，将 JWT 存储到 localStorage
- 应用启动时，从 localStorage 读取 JWT
- 退出登录时，清除 localStorage 中的 JWT

**用户信息**:
- 登录成功后，调用 `/api/app/profile` 获取用户信息、权限、菜单
- 用户信息存储到状态管理
- 权限信息用于按钮显隐控制
- 菜单信息用于动态菜单渲染

2. 调用 `/api/public/oauth/{provider}/authorize` 获取授权 URL
3. 跳转到三方授权页面
4. 用户授权后，三方平台回调到 `/api/public/oauth/{provider}/callback`
5. 后端处理回调，返回登录结果
6. 前端根据结果跳转或提示

**按钮样式**:

**回调处理**:
- 回调页面解析 URL 参数（code, state）
- 调用后端回调接口
- 根据返回结果：
